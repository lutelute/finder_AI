# FinderAI

FinderAIは、ファイル整理とShell・Codex・Claudeを一つのネイティブmacOSウインドウへまとめたワークスペースです。左に場所、中央にファイル一覧、下に開閉・リサイズ可能なTerminalを置きます。標準Finderへ注入・改造・リサイズを行いません。

標準Finder内部へ第三者のTerminalを埋め込む公開APIはありません。別ウインドウをFinderへ重ねる方式も実装済みですが、位置・フォーカス・Spaceの境界を完全には消せません。そのため、幅とライフサイクルが最初から一致する単一ウインドウ構成にしています。旧オーバーレイ版は1.1.0で廃止しました。

## すぐ使う

[最新リリース](https://github.com/lutelute/finder_AI/releases/latest)から`FinderAI.zip`を落とし、`/Applications`へ入れてください。以降は自分で更新を取りに行きます。

インストール済みの場合:

```bash
open "/Applications/FinderAI.app"
```

## アップデート

1日に1度バックグラウンドで確認し、新しいバージョンがあれば**黙ってダウンロードして、アプリを終了したときに入れ替わります**。クリックは要りません。`FinderAI`メニューの「アップデートを確認…」でその場で確認・適用もできます。

強制終了した場合は入れ替わりません。終了時のインストールは正常終了が前提です。

現在実行中のversion・build・commit・実行場所は設定（`⌘,`）の下部で確認できます。`dist`の開発用コピーや、ディスク上だけが新版になって再起動を待っている状態もここで区別します。

更新はEdDSA署名を検証してからでないと適用しません。本番成果物はさらにDeveloper ID署名、Hardened Runtime、Apple notarization、staple、署名済みappcastを必須にします。ローカル開発版だけはフォルダアクセス許可を維持するための自己署名です。どちらの秘密鍵もリポジトリには入りません。

リリースは`./scripts/release.sh <version> [notes.md]`で行います。テスト、Developer ID署名、notarization、staple、ZIP再検証、Sparkle署名、署名済みappcast、GitHubのdraft検査と公開を一続きで行います。ローカル証明書や認証情報不足では公開前に停止します。詳しい準備とGitHub Actionsの秘密情報は[リリース手順](docs/RELEASING.md)にあります。

開発成果物を直接使う場合:

```bash
open "dist/FinderAI.app"
```

Accessibility権限は不要です。デスクトップ・書類・ダウンロードを初めて開いたときは、macOS標準のフォルダアクセス確認が出る場合があります。

## 画面と操作

- 左サイドバー: ピン留め、よく使う項目、場所、よく使うフォルダ、最近。境界をドラッグして160〜360ptで変更できます。
- 上部: 1段目は戻る、進む、親フォルダ、パス欄、再読み込み、新規フォルダ、表示モード。2段目は検索範囲（直下／配下）と検索欄です。
- 表示: リスト、カラム、ギャラリーを直接選べます。リストは4列の比較、カラムは階層移動、ギャラリーは大きなアイコンでの探索向けです。
- 右クリック: 開く、Finderで表示、名前変更、新規フォルダ、ゴミ箱。
- ドラッグ＆ドロップ: 通常は移動、Optionを押しながらならコピー。同名項目は上書きせず、操作前に拒否します。
- 下部: クリックで移動できるパスバー（現在地と祖先）。Terminalは`⌘J`または`TERMINAL`で開閉、上端をドラッグして160〜600ptで変更できます。

### サイドバー

| セクション | 内容 |
|---|---|
| ピン留め | `⌘D`で追加・解除。並びは自分で決められます（最大30件） |
| よく使う項目 | **Finderのサイドバー登録をそのまま表示**します。読み取り専用で、Finderで登録すればここにも出ます |
| 場所 | マウント済みのボリューム。抜き差しで自動更新します |
| よく使うフォルダ | 開いた回数の多い順。2回以上開いたものだけが出ます |
| 最近 | 最後に開いた順 |

同じフォルダは一度しか出ません。ピン留めしたものは、よく使う項目や最近には重複表示されません。空のセクションは見出しごと隠れます。

Finderのよく使う項目が読めない場合は、ホーム・デスクトップ・書類・ダウンロードを代わりに表示します。

### キーボード

| キー | 動作 |
|---|---|
| `⌘N` | 新規ウインドウ（最大20枚、現在のフォルダで開きます） |
| `⌘W` | ウインドウを閉じる |
| `⌘2` | リスト→カラム→ギャラリー表示を順に切替 |
| `⌘⌥S` | 2画面に分割／解除 |
| `⌘L` | 上部のパス欄へフォーカス（全選択。コピーや打ち替えに） |
| `⌘⌥C` | 現在のパスをコピー |
| `Return` / `⌘↓` | 選択項目を開く（フォルダは移動、ファイルは既定アプリ） |
| `Space` / `⌘Y` | クイックルック |
| `⌘D` | サイドバーにピン留め／解除 |
| `⌘⇧.` | 隠しファイルの表示切替 |
| `⌘F` | 検索欄へフォーカス |
| `⌘[` / `⌘]` / `⌘↑` | 戻る／進む／親フォルダ |
| `⌘Z` / `⌘⇧Z` | 取り消す／やり直す |
| `⌘J` | Terminalの開閉 |
| `⌘⌥T` | Terminalセッションの一覧・管理 |
| `⌘,` | 設定（永続セッション、出力ログ） |

最後のウインドウを閉じてもアプリは終了しません。Terminalセッションが動いたままになるためで、Dockアイコンから開き直せます。終了は`⌘Q`です。

検索中に新規フォルダを作成・改名すると検索を解除し、処理した項目を見失わないようにします。削除は必ず確認後にmacOSのゴミ箱へ移し、完全削除は実装していません。

検索範囲の「直下」は表示中フォルダを即時に絞り込み、「配下」は子孫を再帰検索して起点からの相対パスを表示します。配下検索は打ち直し・範囲変更・フォルダ移動で古い処理を中止し、結果は最大5,000件です。カラム表示中に検索すると一時的にリストへ切り替わり、検索を消すとカラムへ戻ります。

名前変更・移動・コピー・新規フォルダは`⌘Z`で取り消せます。取り消しの副作用も上書きを行わず、作成物の取り消しは削除ではなくゴミ箱へ移します。ゴミ箱操作自体の取り消しはFinderの「戻す」に委ねています。

現在のフォルダは監視しており、他のアプリがファイルを追加・削除・改名すると自動で一覧へ反映します。

### 覚える状態

ウインドウ位置・サイズ、サイドバー幅、表示モード、並べ替え列と昇降順、Terminalの高さと開閉、隠しファイルの表示、最後に開いていたフォルダを次回起動時に復元します。最後のフォルダはパスで保存し、消えていればホームから開始します。

これとは別に、全ウインドウのフォルダと実行中セッションの構成を常時スナップショットしており、正常終了しなかった次の起動でだけ復元を提案します（[Terminalの節](#クラッシュしても作業を失わないために)参照）。

## Terminal

フォルダを閲覧しただけではプロセスを開始しません。展開後の`Shell`、`Codex`、`Claude`、または`＋`を押したときだけPTYを作ります。

- Shellは`/bin/zsh -l`です。
- Codex／ClaudeはCLIが見つかる場合だけ有効になります。自動インストールしません。
- セッションはフォルダと種類の組で保持します。別フォルダへ移動しても、既存Terminalへ`cd`や文字を送りません。
- セッションはウインドウをまたいで共有します。同じフォルダを2つのウインドウで開くと、どちらにも同じShellが出ます（2つ起動されません）。
- パネルを隠してもセッションは継続します。
- セッションタブの右クリックでは「タブを隠す（実行は継続）」「現在の表示を記録として保存…」「セッションを終了…」を別々に選べます。タブを隠してもPTYやTerminalバッファは破棄されません。
- セッション終了とアプリ終了は、FinderAI自身が開始したPTYだけを対象にし、実行中なら確認します。

### クラッシュしても作業を失わないために

3つの独立した仕組みがあります。

- **セッション台帳（常時）**: セッションID、種類、フォルダ、backend、時刻、表示・終了状態を`~/Library/Application Support/FinderAI/session-registry.json`へ原子的に保存します。FinderAI再起動後も管理画面に履歴が残ります。Terminalの内容は保存しません。壊れた台帳は同じ場所へ`session-registry.corrupt-*.json`として隔離し、アプリは空の台帳で起動します。
- **構成の復元**: FinderAIが正常終了しなかった場合、次回起動時に「前回の構成（ウインドウとTerminalセッション）を復元しますか？」と確認します。復元は同じフォルダで同じ構成を作り直すもので、通常セッションのプロセスの中身は戻りません。
- **永続セッション（tmux、既定オフ）**: 設定（`⌘,`）の「セッションを永続化（tmux）」を有効にすると、以降のセッションはtmux内で動きます。FinderAIが落ちても終了してもシェルやClaude/Codexは生き続け、同じフォルダの開始ボタンが「◯◯に再接続」になります。tmuxは自動導入しません（`brew install tmux`）。UIからセッションを閉じたときはtmux側も終了します。
- **出力ログ（既定オフ）**: 設定（`⌘,`）の「出力をログに保存」を有効にすると、以降のセッションの出力を`~/Library/Application Support/FinderAI/session-logs/`へ追記保存します。クラッシュ直前に何が起きていたかを後から読めます。14日で自動削除されます。既定でオフなのはプライバシー方針（Terminal内容を保存しない）を守るためです。

### すべてのセッションを俯瞰する

ドロワーに出るのは「今見ているフォルダ」のセッションだけです。ウインドウを何枚も開いて方々でシェルを起こしたときは、表示メニューの「**Terminalセッションを管理…**」（`⌘⌥T`）からセッションセンターを使ってください。

- アプリ内の全セッション（表示中・バックグラウンド・終了）、再起動前の台帳履歴、tmuxに残っているセッションを1つの表に出します（種類・フォルダ・状態）。履歴だけを選んだ場合は「記録を削除」で台帳から整理できます。FinderAI名義tmuxを起動時・再アクティブ時に照合し、生存中なら再接続、確認済みで見つからなければ「消失」と表示します。tmuxへの問い合わせ自体が失敗した場合は消失扱いにしません。
- 「タブに表示」またはダブルクリックで、隠したセッションを元のTerminalバッファごと戻せます。FinderAIをまたいでtmuxに残ったセッションも同じ操作で再接続します。
- 「記録を保存…」はアプリ内セッションの現在の表示バッファだけを、保存先を確認してテキスト化します。自動保存はしません。
- 「フォルダを開く」はセッションの表示状態を変えず、そのフォルダへだけジャンプします。
- 「選択を終了」「すべて終了」で一括終了できます。永続化を切った後にtmuxへ残ったセッションの掃除もここです。
- FinderAI名義（`finderai-`で始まる名前）以外のtmuxセッションには一覧・終了とも一切触れません。
- 検索欄は名前・種類・フォルダ・状態を横断し、「実行中」「非表示」「再接続可能」「履歴」で絞り込めます。任意の記録へ名前を付け、ピン留めして一覧の先頭へ固定できます。最終活動日時も台帳から表示します。

cwdやファイル名をシェル文字列へ連結しません。空白、日本語、引用符、先頭ハイフン、`$()`、改行を含むパスも、URL／`currentDirectory`引数として直接扱います。

## ビルドとテスト

必要環境はmacOS 15以降、Apple Silicon、Swift 6.2系のCommand Line ToolsまたはXcodeです。初回の依存取得だけネットワーク接続が必要です。

```bash
git clone https://github.com/lutelute/finder_AI.git
cd finder_AI
./scripts/run-tests.sh
./scripts/build-workspace-app.sh
./scripts/install-workspace-app.sh   # /Applications へ反映
```

生成物:

- `dist/FinderAI.app`
- `dist/FinderAI.zip`

**ビルドしただけでは`/Applications`のアプリは変わりません。** 普段使うものへ反映するには`install-workspace-app.sh`を実行してください。このスクリプトは`cp -R`ではなく`ditto`を使います。`cp -R`は拡張属性をbundleへ持ち込み、`codesign --verify`が`resource fork, Finder information, or similar detritus not allowed`で失敗するためです。

通常のビルドスクリプトはReleaseビルド、SwiftTermリソース同梱、Info.plist検査、ローカル署名、`codesign --verify --deep --strict`、symlinkを維持したZIP再展開後の再検証まで行います。`FINDERAI_RELEASE=1`ではDeveloper IDとsecure timestampを必須にし、自己署名へフォールバックしません。

### 毎回フォルダアクセスを訊かれる場合

ad-hoc署名のdesignated requirementはcdhashそのものです。

```
$ codesign -d -r- "/Applications/FinderAI.app"
# designated => cdhash H"2fd174cc..."
```

cdhashはビルドごとに変わるため、macOSは各ビルドを別のアプリと見なし、TCCがデスクトップ・書類・ダウンロードの許可を破棄します。これがビルドのたびに「システム設定を開く」が出る理由です。

一度だけ以下を実行すると、requirementが証明書に紐づいて安定し、許可がビルドをまたいで残ります。

```bash
./scripts/create-signing-identity.sh   # ログインキーチェーンへ自己署名証明書を作成
./scripts/build-workspace-app.sh
./scripts/install-workspace-app.sh
```

これはローカル用の証明書をコード署名について信頼する設定であり、あなたのコード署名の信頼設定を実際に変更します。秘密鍵はこのMacから出ず、リポジトリにも入りません。`./scripts/remove-signing-identity.sh`で元に戻せます。

この証明書はDeveloper IDの代わりにはなりません。現在はDeveloper ID証明書を所有していないため、正式公開は意図的に停止します。証明書取得後は[リリース手順](docs/RELEASING.md)に従って認証情報を登録すれば、実装済みパイプラインで署名・公証・公開できます。

現在のテスト範囲と未実施項目は[検証レポート](docs/VERIFICATION_REPORT.md)、実際に触る確認手順は[実機チェックリスト](docs/MANUAL_TEST_CHECKLIST.md)を参照してください。

今回の設計変更、完成時のhash、発見した起動問題、旧session保護、次回の確認順は[2026-07-16開発記録](docs/SESSION_RECORD_2026-07-16.md)に固定しています。

## 安全設計

- Finderプロセスへの注入、private API、AppleScript、Finder再起動を使いません。
- Accessibility、Screen Recording、Input Monitoring、Automationを要求しません。
- ファイル処理は`FileManager`で直接行い、シェルを介しません。
- 移動・コピー・改名は上書きしません。
- フォルダを自分自身やシンボリックリンク経由の子孫へ移動する操作を拒否します。
- テレメトリ、独自ネットワーク通信はありません。Terminal内容の保存は既定で行わず、出力ログはユーザーが表示メニューで有効にした場合だけローカルに残ります。

詳しくは[設計](ARCHITECTURE.md)と[プライバシー／セキュリティ](PRIVACY.md)を参照してください。


## 既知の制約

- 配下検索は最大5,000件で、パッケージ（`.app`など）の内部までは列挙しません。
- Google Drive・OneDrive・DeepLなどがFinderの右クリックへ差し込む項目は出ません。`com.apple.FinderSync`はFinderだけがホストできる拡張ポイントで、第三者のアプリでは構造的に読み込めません。
- 同一フォルダ・同一種類のTerminalセッションは一つです。
- セッションの永続化（tmux）はオプトインで、tmuxを自動導入しません。無効時のセッションはアプリ終了・クラッシュで消えます（構成の復元提案は出ます）。
- File Provider、ネットワークボリューム、非常に大きいフォルダの一覧取得時間は提供元に依存します。一覧取得はUI外で行います。

問題がある場合は[トラブルシューティング](docs/TROUBLESHOOTING.md)を確認してください。

## アンインストール

FinderAIを終了し、`/Applications/FinderAI.app`を削除します。Finderの設定や拡張機能は変更していないため、Finderの再起動や権限削除は不要です。
