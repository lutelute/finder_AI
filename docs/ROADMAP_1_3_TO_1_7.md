# FinderAI v1.3〜v1.7実装計画

## 原則

- PIDではなくFinderAIの安定したセッションIDとtmux名で管理する。
- `finderai-`名義以外のtmuxセッションには触れない。
- Terminal内容の自動保存と外部送信は既定で行わない。
- 各versionを独立PRにし、全テスト、Releaseビルド、必要な実機UI確認後にmergeする。
- 開発成果物をビルドしただけでインストール済みとみなさない。

## v1.3 — 実行中ビルドを識別できる

**完了（PR #5）**

- version、build、Git commit、実行場所を設定画面に表示する。
- Releaseビルド時にcommitをInfo.plistへ埋め込む。
- `/Applications`上のアプリが実行中の版と食い違う場合は再起動を案内する。
- 実際のアプリ名と異なる古いドキュメントを修正する。

## v1.4 — 永続セッション台帳

**完了**

- セッションID、種類、フォルダ、backend、作成日時、最終活動、表示状態をローカルへ原子的に保存する。
- FinderAI再起動後も過去のセッションを管理画面で追跡できる。
- 破損した台帳は隔離して空の状態から起動し、アプリを巻き込まない。

## v1.5 — 照合と復旧

**完了**

- 台帳、アプリ内PTY、FinderAI名義tmuxを起動時とアクティブ化時に照合する。
- 表示中、バックグラウンド、tmux待機、外部接続、終了、消失を区別する。
- 復旧可能なtmuxだけを再接続し、消失した項目は履歴として整理できる。

## v1.6 — セッションセンター

**完了**

- 名前変更、ピン留め、検索、状態絞り込み、最終活動表示を追加する。
- セッション数が増えてもフォルダと種類から素早く目的の作業へ戻れるようにする。
- 終了と非表示を常に別操作として提示する。

## v1.7 — ファイル探索の強化

**実装完了**

- 現在フォルダだけでなく配下を対象にした、キャンセル可能な再帰検索を追加する。
- リスト、カラムに加えてアイコン／ギャラリー表示を追加する。
- File Providerや大規模フォルダでMainActorをブロックせず、古い検索結果を表示しない。

## 完了条件

- v1.7の全機能がmainへmerge済み。
- 全自動テスト、Releaseビルド、署名、zip再展開検証が成功。
- `/Applications/FinderAI.app`へインストールし、version・commitと主要UIを実機確認済み。
